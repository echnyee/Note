## 初始化流程

### 整体流程

![[Pasted image 20260214104500.png]]

- ctor：Entity每次创建时调用一次（迁移不算创建），负责初始化Entity里非存盘属性或者某些仅需要在Entity登录时运行的逻辑
    
- OnFirstCreate：需存盘的Entity（需定义isNew存盘属性），在第一次实例化时调用，负责存盘属性初始化；非存盘Entity可不关注此函数
    
- AfterCtorOrMigrated：Entity创建或者迁移完成后调用，负责一些非属性的初始化或者函数调用，例如注册事件监听等。
    

### 初始化流程拆分原因

- 迁移：Entity迁移时只会打包所有属性到新进程，非属性对象需要重新初始化，事件需要重新注册，因此需要AfterCtorOrMigrated阶段
    
- 性能：打包Entity属性会比重新计算Entity属性开销更小，例如战力计算等
    
- 解耦：如果ComponentA的某个函数调用需要用到ComponentB的初始化数据，在ctor里初始化ComponentB，在AfterCtorOrMigrated里调用ComponentA的函数即可
    

### 自身、基类、Component的ctor顺序

![[Pasted image 20260214104518.png]]

- 先调用基类ctor，然后调用自身ctor，最后再调用Component ctor
    
- 如果基类还有自己的基类，也遵循上述的顺序
    
- 如果Component有基类，先调用基类的ctor，然后才是Component自身的ctor
    
- OnFirstCreate、AfterCtorOrMigrated等也是先基类，后自身再Component的逻辑
    

## 析构流程

![[Pasted image 20260214104531.png]]

- 先调用__before_dtor，然后再调用dtor
    
- __before_dtor和dtor在基类、自身、Component之间的调用顺序和ctor的顺序是相反的
    
- 析构流程拆分为两个阶段是为了解耦
    

## 整体示例

``` lua
DefineClass("Entity")

-- region 基类初始化逻辑
function Entity:ctor()
end

function Entity:AfterCtorOrMigrated()
end

-- endregion 基类初始化逻辑

-- region 基类销毁逻辑
function Entity:__before_dtor()
end

function Entity:dtor()
end
--endregion 基类销毁逻辑

DefineComponent("BuffComponent")

-- region BuffComponent初始化流程
function BuffComponent:ctor()
end

function BuffComponent:__component_OnFirstCreate__()
end

function BuffComponent:__component_AfterCtorOrMigrated__()
end
-- endregion BuffComponent初始化流程

-- region BuffComponent销毁逻辑
function BuffComponent:__before_dtor()
end

function BuffComponent:dtor()
end
-- endregion BuffComponent销毁逻辑

DefineEntity("Avatar", {Entity}, {BuffComponent})
-- region Avatar初始化逻辑
function Avatar:ctor()
end

-- Entity没有OnFirstCreate函数，不需要先调用基类的OnFirstCreate
function Avatar:OnFirstCreate()
end

function Avatar:AfterCtorOrMigrated()
  -- 先基类
  Entity.AfterCtorOrMigrated(self)
  -- 再自身
  -- TODO 初始化逻辑
end
-- endregion Avatar初始化逻辑

-- region Avatar销毁逻辑
function Avatar:__before_dtor()
end

function Avatar:dtor()
end
-- endregion Avatar销毁逻辑
```

- 当继承自Entity时，需要在AfterCtorOrMigrated里手动调用基类的函数，component的函数，框架会帮自动调用
    
- 如果Component有继承时，且希望父类的AfterCtorOrMigrated被调用到，需要在子类里手动调用父类的函数
