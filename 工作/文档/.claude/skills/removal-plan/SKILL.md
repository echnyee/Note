## 计划模板

### 优先级分类
- **P0**：立即清理（安全风险、显著开销、阻断其他工作）
- **P1**：当前迭代内清理
- **P2**：放入待办 / 下次迭代

### 可安全移除项

| 字段 | 详情 |
|------|------|
| **位置** | `path/to/file.lua:line` |
| **理由** | 为什么应该移除 |
| **证据** | 未使用（无引用）、废弃功能开关、已弃用 API |
| **影响** | 无 / 低 — 无活跃消费者 |
| **删除步骤** | 1. 移除代码 2. 移除测试 3. 移除配置 |
| **验证** | 运行测试、检查无运行时错误、监控日志 |

### 延迟移除项（需计划）

| 字段 | 详情 |
|------|------|
| **位置** | `path/to/file.lua:line` |
| **延迟原因** | 有活跃消费者、需要迁移、需相关方确认 |
| **破坏性变更** | 列出任何 API/协议变更 |
| **迁移计划** | 消费者迁移步骤 |
| **时间线** | 目标日期或迭代 |
| **负责人** | 负责人/团队 |
| **验证** | 确认安全移除的指标（错误率、使用计数） |
| **回滚计划** | 发现问题时如何恢复 |

## 移除前检查清单
- [ ] 搜索代码库中所有引用（`grep`、`rg`）
- [ ] 检查动态/反射式调用（Lua 中的 `self[methodName](self, ...)` 模式）
- [ ] 确认无外部消费者（客户端 RPC、NetDefs 协议）
- [ ] 功能开关遥测已审查（如适用）
- [ ] 相关 NetDefs XML 已同步更新
- [ ] 文档已更新
- [ ] 团队已通知（如是共享代码）
