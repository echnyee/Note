name: bug-scaner
description: 当用户要求查找代码中的严重 bug、扫描 bug、排查问题时使用此技能。
---

## 核心原则

**只报告致命和严重问题**：
- **致命** - 数据损坏、可被利用的漏洞、资源重复发放
- **严重** - 关键逻辑错误、状态不一致、资源泄漏

**不报告一般性问题**：边界检查、代码风格、性能优化等常规问题不在扫描范围内。

**注意文件由Perforce管理，如果文件是只读的, 需要先执行 `p4 edit file` 来获取编辑权限。**

## 扫描流程

### 第 1 步：确认核心代码范围

1. **用户提供文件路径**：直接读取指定文件, 如果用户提供的是p4单号, 则通过指令 "p4 describe -S [p4单号]" 获取相关文件改动
2. **用户描述功能**：通过 Grep/Glob 定位相关文件
3. **用户提供 bug 现象**：根据现象反向追踪可能的代码位置

**输出**：列出所有需要扫描的核心文件

### 第 2 步：深度分析核心代码

已知: 相关的代码有一些非常严重的 bug, 你必须把它们找出来, 超过5个问题则只显示最严重的5个问题

### 第 3 步：输出 Bug 报告

```markdown
## Bug 扫描报告

**扫描文件**：X 个核心文件

### 致命问题
1. **[file:line]** 问题描述
   - **影响**：可能导致的后果
   - **修复**：具体修复方案

### 严重问题
2. **[file:line]** 问题描述
   - **影响**：可能导致的后果
   - **修复**：具体修复方案
```

**如果没有发现致命或严重问题，直接说明"未发现严重问题"。**

### 第 4 步：确认修复方案

展示报告后询问用户是否需要修复。未经确认不要主动修改代码。
