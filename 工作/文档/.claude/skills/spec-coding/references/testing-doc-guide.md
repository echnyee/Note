## 2. C7 测试体系

C7 服务端的测试通过 **telnet debug console** 执行 Lua 测试脚本，不使用标准单元测试框架。

### 2.1 测试分层

```
L3: 集成测试（完整玩法流程验证）
    客户端操作 → 服务端处理 → 结果验证
    ──────────────────────────────────
L2: 组件/Service 测试（telnet 调用验证）
    telnet → 直接调用方法 → 断言结果
    ──────────────────────────────────
L1: 函数级测试（telnet 调用验证）
    telnet → 调用单个函数 → 断言返回值
```

### 2.2 测试工具

| 工具 | 用途 |
|------|------|
| telnet debug console | 执行测试脚本、查询运行时数据 |
| `dofile("/Test/xxx.lua")` | 执行测试脚本文件 |
| GM 命令 | QA 快速验证功能 |
| `refreshscript()` | 热更脚本后重新测试 |

## 6. C7 特有测试场景

### 6.1 必须覆盖的固有场景

| 场景类别 | 测试要求 |
|----------|----------|
| **RPC 参数校验** | 所有 CS RPC 参数做非法值、空值、越界测试 |
| **迁移安全** | 操作过程中玩家迁移，回调是否安全 |
| **断线重连** | 操作中途断线，重连后状态是否正确 |
| **热更兼容** | 热更脚本后功能是否正常 |
| **跨天/跨周** | 时间边界的数据重置和状态变更 |
| **配表数据** | 配表 ID 不存在、配表数据边界值 |
| **数值安全** | 负数、溢出、重复领取、并发操作 |
| **SA 日志** | 关键操作是否有完整的审计日志 |

### 6.2 数值安全专项测试

| 测试项 | 说明 |
|--------|------|
| 扣币后余额检查 | 扣币不能使余额为负 |
| 投放限次检查 | 奖励不能超过限次 |
| 幂等性检查 | 重复操作不能重复发放 |
| 顺序检查 | 先扣币再发放，先标记再发放 |

## 7. T/D/B 问题分类

测试失败时，先分类再修复：

| 分类 | 含义 | 修复方式 |
|------|------|----------|
| **T** (Test) | 测试脚本写错了 | 修改测试脚本 |
| **D** (Doc) | 测试文档/设计有误 | 修改测试文档 → 重新生成测试脚本 |
| **B** (Bug) | 业务代码有 Bug | 修改业务代码 |

**分类流程**：
```
测试失败
    ↓
1. Mock/断言/调用方式是否正确？ → T 问题
    ↓
2. 测试设计（预期结果）是否合理？ → D 问题
    ↓
3. 都正确 → B 问题（真 Bug）
```

## 8. 测试原则

| 原则 | 说明 |
|------|------|
| **数据备份恢复** | 每次测试前深拷贝数据，测试后恢复 |
| **自动查找测试数据** | 从 TableData 自动查找可用参数，不硬编码 |
| **错误捕获** | 用 `custom_xpcall` 包裹被测函数，防止异常中断 |
| **独立运行** | 测试用例之间互不依赖 |
| **结果汇总** | 输出通过/失败数量和失败详情 |
