# 代码质量检查清单（C7 服务端适配版）

## 错误处理

### 需标记的反模式

- **吞没异常**：空的错误处理或仅打日志不处理
  ```lua
  -- 危险：静默失败
  local ok, err = pcall(func)
  -- err 被忽略
  ```
- **过于宽泛的捕获**：捕获所有异常但未区分类型
- **错误信息泄露**：内部细节暴露给客户端
- **缺少错误处理**：对可能失败的操作（IO、网络、解析）未做保护
- **异步错误处理**：Service 回调中未检查 `code ~= NO_ERR`

### 最佳实践

- [ ] 错误在适当边界被捕获
- [ ] 错误消息对用户友好（不暴露内部细节）
- [ ] 错误日志包含足够的调试上下文
- [ ] 异步错误被正确传播或处理
- [ ] 可恢复错误定义了降级行为
- [ ] 关键错误触发告警/监控

### 排查提问
- "当这个操作失败时会发生什么？"
- "调用方是否知道出了问题？"
- "是否有足够上下文来调试这个错误？"

## 性能与缓存

### 高开销操作

- **热路径上的昂贵操作**：循环内的正则编译、JSON 解析
- **不必要的重复计算**：相同计算执行多次
- **缺少记忆化**：纯函数使用相同参数反复调用

### 数据库与 I/O

- **N+1 查询**：循环中逐条查询而非批量
  ```lua
  -- 差：N+1
  for _, id in ipairs(ids) do
      local data = db:findOne({ _id = id })
  end
  -- 好：批量查询
  local dataList = db:find({ _id = { ["$in"] = ids } })
  ```
- **缺少索引**：在未建索引的字段上查询
- **过度获取**：查询整个文档但只用到几个字段
- **无分页**：将整个数据集加载到内存

### 缓存问题

- **缺少缓存**：对昂贵操作的结果未缓存
- **缓存无 TTL**：过期数据永久提供
- **缓存未失效**：数据更新但缓存未清除
- **缓存键冲突**：键唯一性不足

### 内存

- **无界集合**：数组/表无限增长
- **大对象持有**：持有引用阻止 GC
- **循环内字符串拼接**：应使用 `table.concat`
- **加载大文件到内存**：应使用流式处理

### 排查提问
- "这个操作的时间复杂度是什么？"
- "数据量 10x/100x 时表现如何？"
- "这个结果可缓存吗？应该缓存吗？"
- "能否批量处理而非逐个？"

## 边界条件

### Nil 处理

- **缺少 nil 检查**：访问可能为 nil 的对象属性
  ```lua
  -- 危险：未检查 nil
  local name = player.profile.name  -- profile 可能为 nil

  -- 安全
  local name = player.profile and player.profile.name
  ```
- **Truthy/Falsy 混淆**：`if value then` 当 `0` 或 `false` 是合法值时
- **nil 与不存在的不一致**：混合使用 nil 和不存在

### 空表处理

- **空表未处理**：代码假设表有元素
- **首/末元素访问**：`arr[1]` 或 `arr[#arr]` 未检查长度
- **遍历空表**：`pairs`/`ipairs` 安全，但 `next` 判断需注意

### 数值边界

- **除以零**：除法前未检查除数
- **整数溢出**：大数超出安全范围（32 位 int 最大 2147483647）
- **浮点比较**：使用 `==` 而非 epsilon 比较
- **负值**：不应为负的索引或计数
- **Off-by-one 错误**：循环边界、数组切片、分页

### 常见危险模式

```lua
-- 危险：nil 访问
local name = user.profile.name

-- 危险：空表访问
local first = items[1]

-- 危险：除零
local avg = total / count

-- 危险：truthy 检查排除合法值
if value then end  -- 0 和 false 会被跳过
```

### 排查提问
- "如果这个值是 nil 会怎样？"
- "如果这个集合为空会怎样？"
- "这个数字的合法范围是什么？"
- "在边界值（0、-1、MAX_INT）时会发生什么？"
