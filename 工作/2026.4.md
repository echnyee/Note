## 类型概念
## 1) 基础类型（concrete type / 具体类型）

像这些都是**具体类型**（不是接口）：
- 基础：`int`, `string`, `float64`, `bool`
- 复合：`struct`, `[]T`, `map[K]V`, `*T`
- 你自定义的命名类型：`type MyInt int`

具体类型的变量里放的是“具体值”，例如：
Go
```
var x int = 3
```
---

## 2) 接口类型（interface type）

接口类型定义了一组“方法集合”（method set），例如：

Go
```
type error interface { Error() string }
type Stringer interface { String() string }
```
以及空接口：

Go
```
interface{} // 或 any
```
- `interface{}` 没有任何方法要求，所以**任何具体类型**都实现它。

---

## 3) 静态类型（static type）

**静态类型**就是“变量在编译期声明的类型”。

例如：

Go
```
var i interface{} = "hello"
```
这里 `i` 的静态类型是 `interface{}`（编译器看到 i 就把它当接口来处理）。

再如：
Go

```
var err error = ErrNegativeSqrt(-2)
```

这里 `err` 的静态类型是 `error`。

静态类型决定了：
- 你能对变量调用哪些方法/做哪些操作（编译期检查）
- 赋值/传参是否类型匹配

---

## 4) 动态类型（dynamic type）与动态值（dynamic value）

当一个变量的静态类型是 **接口 时，它在运行时会装着某个具体值。此时接口值可以理解为一对：
- **动态类型**：里面装的具体类型是什么（例如 `string`、`*MyError`）
- **动态值**：那个具体值本身（例如 `"hello"`、`&MyError{...}`）
例子：
Go
```
var i interface{} = "hello"
```
- 静态类型：`interface{}`
- 动态类型：`string`
- 动态值：`"hello"`

你可以用：
Go
```
fmt.Printf("%T\n", i) // 打印动态类型：string
fmt.Printf("%v\n", i) // 打印动态值："hello"
```
## 初始化速记
- **slice**：能用 `var s []T; append` 就别急着 `make`；能估计长度就 `make([]T, 0, n)`
- **map**：要写就 `make` 或字面量；`var m map[...]...` 只适合延迟分配/只读
- **二维 slice**：不规则直接逐行 `make`；规则矩阵也逐行；性能敏感用连续内存
- **struct**：字段初始化用 `T{}` 或 `&T{}`，默认配置用零值
- **泛型**：`Type[Concrete]` 实例化；泛型函数多依赖推导；要比较就 `comparable`
- **io**：优先用 `io.Copy` 等组合函数；写 wrapper 按固定三步走

## GC
几个简单的结论
1. GoGC很多时候是cpu和内存的平衡，给的内存多，gc频率低，cpu友好，给的内存少，内存友好，cpu消耗高
2. mark-sweep，从root开始walkthrough，也叫scan，找到不可达的内存，进行可分配标记，是非移动型的GC，有些GC在gc过程中会移动内存，让内存都放在一起。go不会
3. GC成本
	1. 主要成本（dominant）：**marking & scanning（标记与扫描）**，并且和 **live heap 大小、指针密度/结构**强相关：指针越多、结构越“难并行”(比如链表/树深)，单位字节扫描成本越高。
	2. 固定成本：每轮初始化/准备之类的小开销
	3. sweep 成本被文档认为相对小，在 Go 当前实现里通常远小于 mark/scan
4. 通过root节点开始scan，就不会有引用计数的循环引用问题，例如最经典的A<->B互相引用，在他们释放之后，引用计数仍然>0，而在root scan的流程中，如果A和B释放之后（仅存在互相的引用），那么从root那边scan下来，就会断开了，会被认为是不可达的。
